## Introduction

KubeFleet is a cloud-native solution for multi-cluster application management. Use KubeFleet to intelligently schedule workloads, roll out changes progressively, and perform administrative tasks easily, across a group of Kubernetes clusters on any cloud or on-premises.

KubeFleet delivers a set of Kubernetes controllers and Custom Resource Definitions (CRDs) which provide an advanced cloud-native solution for multi-cluster application management.

Use KubeFleet to schedule workloads intelligently, roll out changes progressively, and perform administrative tasks easily, across a group of Kubernetes clusters in any location.

Visit [our website](https://kubefleet.dev/) for information about KubeFleet.

## Community meetings

Every month we host a community call in US/EU and ANZ/Asia time zones to showcase new features, review key open issues, and engage in a Q&A. For community calls, anyone from the KubeFleet community can participate, present a topic, or host. All are welcome!

The two meetings will be identical, so if you miss one and the recording isn't out, you can always catch the other one! If you miss both, you can always catch later by watching the recordings below.

## How can I get involved?

One of the easiest ways to contribute is to participate in discussions at community engagements or write a blog entry.

If you're looking for something to work on, read the [contribution guidelines](https://github.com/kubefleet-dev/kubefleet/blob/main/CONTRIBUTING.md) and then you start by looking for GitHub issues, marked with `good first issue` or the `help wanted` labels:

- [KubeFleet repo](https://github.com/kubefleet-dev/kubefleet/issues)
- [Website and documentation repo](https://github.com/kubefleet-dev/website/issues)
- [Cookbook repo](https://github.com/kubefleet-dev/kubefleet-cookbook/issues)

And, we can always use more testing, have more and improved docs, or just write a blog post on what you have discovered whilst using KubeFleet.
